# Helm Operating Constitution v1

## 状态

这是 Helm 当前第一轮正式 operating constitution。
它不是品牌宣言，也不是抽象价值观集合，而是后续页面 judgement、handoff、settings boundary、self-check 和 acceptance 文档可以直接引用的 operating charter。

## Helm 是什么

Helm 是一个 `workspace-first`、`membership-backed`、`judgement-first` 的组织级 operating workspace。

它当前要做的是：

1. 把最重要的经营判断放到前台
2. 把会议、机会、审批、记忆和后续动作接成同一条 operating loop
3. 让 AI recommendation、review、handoff、worker 使用、audit 和 replay 都保持可追溯
4. 让 trial、payment、seat、worker entitlement 和 lifecycle truth 继续挂在同一套组织边界上
5. 让 Helm 团队开始真的在 Helm 上经营 Helm

## Helm 不是什么

Helm 当前不是：

1. 通用聊天产品
2. 完整战略平台
3. 完整 OKR / KPI 平台
4. 完整 org chart / HR 平台
5. 完整 knowledge platform
6. 完整 workflow / orchestration engine
7. 完整 billing platform
8. finance console
9. 完整 company operating system 平台

## Helm 追求什么结果

当前默认追求 5 类结果：

1. 让组织知道现在最该推进什么
2. 让 recommendation 不越过 commitment boundary
3. 让 handoff、review、approval、memory、audit 可以解释“为什么现在这样判断”
4. 让 trial / active / grace / read_only 的 lifecycle truth 保持 narrow、honest、可解释
5. 让 internal operating workspace 服务“让 Helm 团队在 Helm 上经营 Helm”

## Helm 拒绝什么做法

Helm 当前明确拒绝：

1. 用 provider state 直接替代 workspace 自己的 lifecycle truth
2. 把 recommendation 包装成 commitment
3. 把 review-before-send、non-commitment、discussion-only、boundary-only 藏起来
4. 把 token / storage / retrieval 变成 customer-facing 单独计费项
5. 把 trial 做成功能阉割版
6. 把 settings 扩成 finance console
7. 把 internal operating workspace 退回对象入口堆叠
8. 把 foundation 写成空话却不影响页面和守线

## 当前默认判断优先级

当前默认 judgement priority 冻结为：

1. 先看 workspace 当前最重要的经营压力是什么
2. 再看当前 recommendation 是否已经满足 boundary / prerequisite / dependency
3. 再决定它只是 discussion、boundary review，还是已经进入可执行 rail
4. recommendation 不等于 commitment，外部表达继续默认 review-before-send
5. 如果 lifecycle 进入 grace / read_only，只收窄新的高成本 processing，不把产品整体阉割
6. replay、audit、memory linkage、source-use ledger 必须能解释当前判断

## 当前不能轻易改写的 product truth

以下 truth 在 v1 中属于 constitution 级别：

1. `workspace-first`
2. `membership-backed`
3. `controlled-trial`
4. `recommendation 不等于 commitment`
5. `discussion-only / boundary-only / review-before-send / non-commitment` 必须显式可见
6. trial / paid 都开放完整当前核心功能
7. token / storage / retrieval 不对外单独计费
8. lifecycle-based guard 必须 narrow、honest、可解释
9. replay / audit / memory / handoff 必须可追溯
10. internal operating workspace 服务“让 Helm 团队在 Helm 上经营 Helm”

## 这层如何影响产品

这份 constitution 当前会被以下入口直接引用：

1. dashboard：解释 Helm 为什么先把经营 judgement 放在前台
2. settings / billing overview：解释 lifecycle、seat、worker、payment truth 为什么这样组织
3. internal operating workspace：解释 Helm 团队为什么要先看经营判断，而不是重新扫对象列表
4. unified detail navigation / handoff：解释 detail 为什么现在可以代表工作区继续说话
5. self-check / boundary checks：防止 foundation 被写成抽象口号或平台空壳
6. acceptance docs：说明当前 operating truth 是如何约束 billing / lifecycle / operating surfaces 的

## 当前刻意未做

本轮刻意未做：

1. 完整战略管理平台
2. 完整知识平台
3. 完整组织平台
4. 完整 workflow engine
5. runtime sandbox

这层当前只是 Helm 第一轮 operating constitution foundation，不是完整 operating platform。
