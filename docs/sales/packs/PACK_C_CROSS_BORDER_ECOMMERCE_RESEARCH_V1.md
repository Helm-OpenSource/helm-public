# Pack C — 跨境电商运营 行业 Pack 调研、定位与需求 V1（候选）

> **重要：** Pack C 在选型调研中标记 ★ 不确定性。Helm 主战场是"客户会议后推进链"，跨境电商工作流可能更靠近"日常运营轮班"。Pack C 在 1-2 家访谈完成前**仅作候选**，未冻结实施。

## 1. 适用 ICP
跨境电商运营公司 / 品牌出海团队 / 跨境电商服务商。

**对终端客户：** 让你的跨境运营团队在每周复盘 + 每日例会后 5 分钟内，知道选品、广告、合规、客服哪条线该怎么推、谁来推、不能轻易承诺什么。
**对自媒体：** 把"跨境运营每日例会→该做什么"变成可截图、可审计、可交接的推进链。

## 2. 行业调研（候选）
### 2.1 客群画像
| 维度 | 描述 |
|---|---|
| 公司规模 | 30-300 人跨境电商 / 品牌出海 |
| 业务模式 | 自有品牌 + 多平台运营（亚马逊、Shopify、TikTok Shop、Temu） |
| 年 GMV | ¥3,000 万 - ¥5 亿 |
| 决策买家 | 创始人 / 运营总监 / 选品负责人 / 广告负责人 |
| 工具栈 | 平台后台 + 广告平台 + 客服系统 + 飞书/钉钉/Slack + ERP |
| AI 现状 | 选品/广告/客服自动回复已普遍用 AI，缺跨工具运营推进闭环 |

### 2.2 痛点（候选——待访谈验证）
1. 每日例会效率低——信息碎、跨平台数据散，会后不知道做什么
2. 跨平台决策割裂——亚马逊/Shopify/TikTok Shop 协同难
3. 广告与选品反馈链长——投放→选品→补货 handoff 慢
4. 合规风险滞后发现——平台规则变化、IP 投诉、税务滞后
5. 客服与运营 handoff 断层
6. 运营 SOP 不沉淀——好运营离职经验不复用

### 2.3 关键不确定性 ★
| 问题 | 假设 | 验证 |
|---|---|---|
| 是否有"客户会议"？ | 少量与平台经理/服务商会议；主要内部运营会 | ★ |
| 内部会议能否进 Helm 链？ | 可——把例会 + 周复盘作触发点 | ★ |
| 内部会议的"承诺/复核"逻辑成立？ | 成立——选品决策、广告预算、合规整改是承诺型 | ★ |
| 会被船长 BI / 店小秘等替代？ | 不会——Pack C 是横跨工具的推进闭环 | ★ |

**验证路径：** 3-5 家跨境电商客户访谈（见 §11）。

### 2.4 现有工具栈盲区
| 工具 | 解决 | 没解决 |
|---|---|---|
| 平台后台 | 销售、库存 | 跨平台运营推进 |
| 广告平台 | 投放、转化 | 投放→选品→补货反馈链 |
| 跨境电商 SaaS（船长 BI 等） | 多平台数据聚合 | 例会→推进链、SOP 复盘 |
| ERP | 订单、库存 | 决策链路与承诺审计 |
| 客服系统 | 工单 | 工单→运营/选品 handoff |

### 2.5 竞品对比
| 类型 | 代表 | 与 Pack C 差异 |
|---|---|---|
| 跨境电商 SaaS | 船长 BI、店小秘、领星 | 数据聚合；Pack C 推进闭环 |
| AI 选品 | Helium 10、Jungle Scout | 选品建议；Pack C 横跨选品/广告/合规/客服 |
| AI 广告投放 | Smartly、平台原生 | 投放优化；Pack C 把结果接到运营推进 |
| 通用 Agent 平台 | 扣子/百炼 | 是 builder；Pack C 是行业成型方案 |

### 2.6 开源传播
**适合，但形态需适配。** 跨境自媒体生态发达；运营老板有"可截图、可分享"展示欲望；与 Pack A/B 客群完全不重叠。**约束：** 跨境老板技术评估能力低于 SaaS/SI，开源叙事传播力低于 Pack A/B；客单价低于 A/B。

## 3. Pack C 定位（候选）
### 3.1 主卖点
1. 每日运营推进——例会→选品/广告/合规/客服该做什么
2. 跨平台决策协同——亚马逊/Shopify/TikTok Shop 同步
3. 合规与风险面板——平台规则、IP 投诉、税务 readout
4. 客户反馈闭环——客服工单→运营推进 handoff

### 3.2 No-Go（候选）
团队 < 10 人，无跨平台协同；客单价 < ¥3 万；白牌铺货模式；单一平台无协同。

## 4. Skill 设计（候选）
> 基于假设的初步设计，待访谈定稿。

### 4.1 Skill C1：每日运营推进清单
- 触发：例会结束 / 09:30 自动推送
- 输入：各平台昨日数据 + 广告 + 客服工单 + 例会纪要
- 输出：当日推进清单（选品/广告/合规/客服四线）
- 复核点：清单默认建议；运营总监确认后分发

### 4.2 Skill C2：跨平台决策协同
- 触发：平台间数据冲突 / 一个平台决策影响其他
- 输入：多平台数据 + 历史决策
- 输出：跨平台决策建议 + 影响测算
- 复核点：默认建议；运营总监决策

### 4.3 Skill C3：合规与风险 Readout
- 触发：平台规则更新 / IP 投诉 / 税务 / 客服合规风险
- 输入：规则订阅 + 工单 + 历史合规事件
- 输出：风险 readout（类型/影响/整改建议）
- 复核点：整改决策由合规/创始人拍板

### 4.4 Skill C4：客户反馈→运营 Handoff
- 触发：客服工单聚合（每周/主动）
- 输入：全部工单 + 退货 + 评论分析
- 输出：反馈聚合 Pack（产品质量/广告差距/平台体验）+ 行动建议
- 复核点：客服主管 + 运营总监共同确认

### 4.5 开源/商业边界
| Skill | 开源 | 商业版（Pack C） |
|---|---|---|
| 通用 Meeting-to-Action / Handoff worker | ✅ basic | — |
| 跨境电商行业适配（平台连接器、运营术语） | ❌ | ✅ Pack C certified |
| 跨平台数据聚合 | ❌ | ✅ Pack C certified |

## 6. 数据接入
### 6.1 最小源
| 源 | 用途 | scope |
|---|---|---|
| 平台后台（亚马逊 SP-API、Shopify API、TikTok Shop API） | 销售、库存、评论 | API 只读 |
| 广告平台（Meta、Google、TikTok Ads） | 投放数据 | API 只读 |
| 客服系统（Zendesk / 独立站工具） | 工单 | API 只读 |
| ERP（旺店通/聚水潭/自建） | 订单、库存 | 只读 + 双向（可选） |
| IM（飞书/Slack/钉钉） | 例会纪要 | OAuth + 只读 |

### 6.2 接入复杂度
| 系统 | 复杂度 | 工程师 |
|---|---|---|
| 标准平台 API（SP-API、Shopify） | 中 | L1/L2 |
| 多店铺/多账号 | 中高 | L2 |
| 自建 ERP / 老旧 | 高 | L2/L3 |
| 多事业部/多品牌 | 高 | L3 |

服务空间为"中"——平台 API 标准化较高，但跨平台 + 多账号 + 自建 ERP 仍需定制。

### 6.3 私有化
跨境团队对私有化需求低，**Pack C 默认 SaaS**。

## 7. 不做清单（红线）
| 不做 | 理由 |
|---|---|
| 自动调广告预算 / 改价 | 商业决策不能 AI 替代 |
| 自动回复客服工单 | 客户关系不能 AI 替代 |
| 自动下单补货 | 库存决策必须人工 |
| 自动判定"是否合规" | 合规判定需人工/法务 |
| 替运营总监做"砍 SKU" | 不替代管理判断 |

**对外措辞红线：** 不得用"AI 替你运营"、"自动管理跨境店铺"、"无需运营总监"。

## 10. 风险
1. **★ 关键：契合度未验证**——工作流是否真适配主战场。**对策**：3-5 家访谈，未通过则替换工业 SaaS
2. 平台 API 政策风险——亚马逊/Shopify 可能受限。**对策**：架构预留多 connector 替换
3. 客单价低 → 工程师投入意愿不足——更愿服务 A/B。**对策**：实施流程标准化，L1 工程师 3-5 天可接入
4. 行业波动大——平台政策与汇率影响付费意愿。**对策**：早期签 quarterly 合同
5. 首批 design partner ≥2 家是否就位——**待补充**
