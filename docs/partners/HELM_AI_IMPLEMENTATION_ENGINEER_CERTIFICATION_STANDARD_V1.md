# Helm 认证 AI 实施工程师标准 V1

## 1. 目的
为三个行业 Pack 的企业落地提供可规模化的人才供给：Helm 出标准与资格认定，认证流程公开透明，市场机制决定服务价格、派单、独家与否。

**核心原则（创始人裁决）：** Helm 制定标准并出具资格认定，其余放开，以充足市场供给加快社会实施能力建设。

## 2. 范围与边界
### 2.1 Helm 做什么
| 类别 | 做 |
|---|---|
| 标准 | 出具能力素质要求、考试大纲、行业 Pack 实操规范 |
| 评测 | 笔试题库、实操沙箱、案例提交评审 |
| 资格 | 颁发证书（L1/L2/L3 三级），公开注册名录 |
| 工具 | 认证工程师专属沙箱、Pack 文档、技术答疑通道 |
| 撤销 | 行为不当者撤销资格并公示 |

### 2.2 Helm 不做什么（明确放开）
| 类别 | 不做 | 理由 |
|---|---|---|
| 服务定价 | 不规定 | 市场决定 |
| 派单与分成 | 不做集中派单、不抽成 | 避免与工程师抢饭吃 |
| 独家 | 不要求 | 市场化 |
| 雇佣 | 不雇佣、不签代理 | 工程师是独立第三方 |
| 客户合同 | 与 Helm 无关 | Helm 不在交易链路 |

**三方关系：** Helm ↔ 工程师（资格关系）；Helm ↔ 客户（软件许可关系）；工程师 ↔ 客户（独立服务合同）。互不交叉。

## 3. 认证分级
### 3.1 三级体系
| 级别 | 名称 | 门槛 | 可承接 |
|---|---|---|---|
| L1 | Helm Certified Implementer (HCI) | 笔试 + 1 个沙箱实操案例 | 开源版部署、基础连接器、Pack 标准安装 |
| L2 | Helm Certified Senior Implementer (HCSI) | L1 + 2 个真实案例评审 + 高级笔试 | 定制连接器、Pack 客户化、企业资源对接 |
| L3 | Helm Certified Industry Expert (HCIE) | L2 + 某 Pack ≥3 个落地 + 行业方案评审 | Pack 深度定制、复杂集成、design partner |

L3 必须挂靠某一 Pack（L3-A / L3-B / L3-C），允许同时持有多个 L3。

## 4. 认证流程
### 4.1 注册（任何人可申请）
公开入口 `certify.helm.dev`。要求实名 + 邮箱/手机 + GitHub。**Helm 不审核背景**——是否能拿证由考试与案例决定。

### 4.2 L1
注册 → 学习材料（公开免费）→ 笔试（60 题/90 分钟，70 分通过，季度更新题库）→ 沙箱实操（5 工作日内出结果）→ 颁证 → 公开名录。证书 2 年有效，续期 30 题笔试。

### 4.3 L2
L1 满 6 个月 → 提交 2 个真实案例（脱敏）→ 高级笔试 → 评审会（3 名 L3 + 1 名 Helm 核心）→ 颁证。否决率预期 30-40%。

### 4.4 L3
L2 满 12 个月 → 某 Pack ≥3 个落地 → 行业方案评审（提交"Pack 演进建议"，需被接受为 PR 候选）→ 颁证。每 Pack 每年限额 ≤15 人。

## 5. 行为准则与撤销
### 5.1 红线（违反即撤销）
1. 冒用 Helm 官方身份对外
2. 模糊"Helm 软件"与"实施服务"合同主体
3. 把 Helm 商业版 SLA / 审计 / 合规承诺扩展给客户
4. 客户数据搬运无书面授权
5. 考试或案例作弊
6. 未授权使用 Helm 商标做商业获客

### 5.2 黄线（警告 + 二次违反撤销）
1. 公开物料未注明"独立服务方"身份
2. 案例公示未脱敏
3. 合同未明确"软件由 Helm / 服务由本人"分层

### 5.3 撤销公示
7 工作日内在公开名录公示（姓名、证书号、级别、简要事由）。

## 6. 公开名录
**展示：** 姓名、证书编号与级别、有效期、Pack 专长（L3）、脱敏案例摘要、自愿公开的联系方式。
**不展示：** 价格、评分、排名、接单状态。
**不是派单平台**——Helm 不撮合交易。

## 7. 与三个 Pack 的协同
| Pack | 工程师角色 | 客户付费链路 |
|---|---|---|
| Pack A（B2B SaaS） | CRM/IM/邮箱接入 + 销售流程客户化 | 客户付 Helm 年费 + 付工程师服务费（两条独立合同） |
| Pack B（SI） | SI 公司员工承接交付项目 | 同上 |
| Pack C（跨境电商，待定） | 店铺/广告/客服系统接入 | 同上 |

**关键边界：** 工程师不能 fork 或私改 Pack 代码对外销售（触发红线 1）。

## 8. 启动节奏
| 阶段 | 时间 | 里程碑 |
|---|---|---|
| 标准制定 | W1-2 | 本文件 + 题库 + 沙箱 |
| 内测 | W3-4 | 5-10 名种子 L1 内测 |
| 公开发布 | W5 | certify.helm.dev 上线 + 公开课 + 公众号传播 |
| 首批扩展 | W6-12 | L1 ≥30 / L2 ≥10 / L3 ≥3 |

**Helm 投入：** 0.5 FTE 持续。**不需要：** 派单系统、销售团队、服务营销。

## 9. 风险
1. L2 案例评审瓶颈：早期 L3 不足，由 Helm 核心代评，6 个月后进入"L3 评 L2"稳态
2. 认证沦为门面：题库泄漏/案例造假——题库季度更新 + 抽样回访客户
3. 品牌冒用：法务侧准备维权模板
4. 市场化与质量平衡：客户侧暴露 L1/L2/L3 分级让客户自选
5. 海外认证：本期仅中国，海外延 ≥6 个月
