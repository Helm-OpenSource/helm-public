# Helm Investor Narrative CN v1

## 1. What Wedge Helm Owns Today

Helm 当前最清楚拥有的是 **meeting-first operating wedge**。

这条 wedge 不只是会议记录，而是已经把：

- dashboard
- meeting detail
- approvals
- memory
- diagnostics

收成了一套连贯的经营协同 loop。

## 2. How Helm Expands From Meeting Wedge to Broader Operating System

Helm 的扩张路径不是从“通用 AI 平台”往下落，而是从高频工作流入口一步步往上长：

1. 先拿会议等高频入口
2. 把会议之后的推进链做清楚
3. 把过程沉淀成经营记忆
4. 把 AI worker 与 human review 的协同关系做清楚
5. 把治理边界和可控性做成长期信任基础

这条路径比“先讲平台、再找场景”更窄，但也更有产品抓手。

## 3. Why Memory and Governance Matter

如果只有入口，没有记忆，系统就会退化成另一个 AI 助手。

如果只有入口和记忆，没有治理，系统就无法从单点工具走向团队长期使用。

所以 Helm 的长期壁垒在于：

- operating memory
- formal review
- controllable governance

这三层让它更像 business collaboration operating system，而不是一次性 AI output 工具。

## 4. Why Helm Is Not Trying to Be a Generic AI Platform

Helm 不把自己讲成通用 AI 平台，原因很直接：

- generic platform language 容易失去场景抓手
- 也容易 overclaim 当前能力
- 对企业客户来说，推进质量、复核和边界比“平台感”更重要

所以 Helm 选择的是：

- 从高频工作流切入
- 把目标推进做深
- 把记忆和治理做厚

## 5. What Has Already Been Proven

当前已经证明的内容应诚实限制在：

- meeting-first operating wedge 已经可读
- dashboard / meeting / approvals / memory / diagnostics 的 loop 已经成立
- recommendation / review / governance posture 已有明确边界
- trial / billing lifecycle truth 已经清楚
- current China commercial rail 已有 narrow truth
- China-market messaging 已能把 Helm 从 meeting tool 抬到 AI 经营协同操作系统

## 6. What Remains to Be Proven

仍需继续证明的，不应提前讲成已完成：

- 更广的 connector strengthening
- 更深的 multi-role operating workspace adoption
- 更广的 AI worker collaboration proof
- 更强的 paid conversion and expansion motion
- 更系统化的 partner / ecosystem leverage

## 7. Investor Closing

Helm 当前最值得关注的，不是它是不是“又一个 AI 工具”，而是它是否能从最强的高频入口出发，把推进、记忆、复核和治理收成一套越来越难替代的 business collaboration operating system。
