# Pack A — B2B SaaS 销售推进 行业 Pack 调研、定位与需求 V1

## 1. 适用 ICP
B2B SaaS / 企业软件公司（与 ICP playbook ICP 1 对齐）。

**对终端客户：** 让你的销售团队在每次客户会议结束 5 分钟内，知道该跟什么、谁来跟、怎么说、哪些不能承诺、哪些需要管理者介入。
**对自媒体：** 把"会议结束→该做什么"变成可截图、可审计、可交接的推进链。

## 2. 行业调研
### 2.1 客群画像
| 维度 | 描述 |
|---|---|
| 公司规模 | 30-500 人 B2B SaaS，年收入 ¥1,000 万 - ¥5 亿 |
| 销售团队 | 5-50 名销售 + 售前 + CS |
| 客单价 | ¥10 万 - ¥500 万 ARR |
| 销售周期 | 30-180 天 |
| 决策买家 | 创始人 / CEO / COO / 销售 VP / RevOps |
| 工具栈 | CRM（销售易/纷享/SFDC 中国）+ 飞书/钉钉/企微 + 邮箱 + 项目管理 |
| AI 现状 | 试过 1-2 个 AI 工具，缺跨工具的推进闭环 |

### 2.2 痛点
1. 会后漏跟进——任务散落 IM / CRM 备注 / 个人脑子
2. 管理者盲区——VP 看不到"今天哪几个会需要介入"
3. 承诺越界——口头答应"我们能做 X"，交付侧不知情
4. CS Handoff 丢上下文
5. 机会判断不透明
6. 跟进话术缺乏复用

### 2.3 现有工具栈盲区
| 工具 | 解决 | 没解决 |
|---|---|---|
| CRM | 商机记录、管线 | 会后 5 分钟跟进闭环、承诺审核 |
| 会议纪要 | 录音转写、摘要 | 摘要变行动 |
| AI 销售助手 | 提示语、邮件草稿 | 跨工具推进、管理者关注、承诺/复核 |
| IM | 沟通 | 推进闭环、审计 |

**差异化定位：** 填补"会议→推进链"断层；不替代 CRM 或纪要工具，是把它们的产出收成可推进、可复核、可交接的链路。

### 2.4 竞品对比
| 类型 | 代表 | 与 Pack A 差异 |
|---|---|---|
| 会议纪要 + AI | 飞书妙记/腾讯会议/Otter | 止于摘要 |
| AI 销售助手 | Gong/SFDC Einstein/销售易 AI | 围绕 CRM；Pack A 横跨 CRM+IM+邮箱+会议 |
| 通用 Agent 平台 | 字节扣子/阿里百炼/钉钉悟空 | 是 builder；Pack A 是行业成型方案 |
| RevOps 平台 | Clari/Gong RevOps | 重报表；Pack A 重推进 |

### 2.5 开源传播
**适合且优先级最高。** 销售 SaaS 是中国 To B 自媒体热议题；CTO/CIO 评估开源工具意愿强。

## 3. Pack A 定位
### 3.1 主卖点
1. Meeting-to-Action Pack——会后 5 分钟跟进清单
2. Revenue Operating Loop——管线推进与机会风险
3. Manager Attention——VP 的"今天该看的几个会"
4. CS Handoff——交付/CS 接手的上下文 Pack

### 3.2 No-Go
客户只想买纪要 / 团队没有协同链 / 客单价 < ¥5 万。

## 4. Skill 设计（4 个）
### 4.1 Skill A1：会议跟进清单
- 触发：客户会议结束（飞书/钉钉/腾讯会议）
- 输入：录音/纪要 + CRM 商机 + 历史记忆
- 输出：跟进清单（谁/做什么/何时/措辞建议/不能承诺什么）
- 复核点：跟进措辞默认建议；对外发送前需销售确认
- 内核：Meeting-to-Action basic worker（开源） + 商业版 follow-up draft governor

### 4.2 Skill A2：今天该跟的 5 个客户
- 触发：每日 09:00 / 销售唤起
- 输入：活跃商机 + 7 天活动 + 未完成跟进
- 输出：当日 5 个优先客户 + 风险标注 + 建议动作
- 复核点：排序是建议；销售可调整
- 内核：Opportunity Judge shadow worker（开源 basic） + 商业版 priority scorer

### 4.3 Skill A3：销售主管会议风险面板
- 触发：每日 / 主管打开
- 输入：全部会议 + 历史承诺 + 客户健康度
- 输出：需介入的 2-3 个会（原因 + 建议时机）
- 复核点：介入是建议；主管自主决定
- 内核：Manager Attention worker（商业版）

### 4.4 Skill A4：交付/CS Handoff Pack
- 触发：销售标记成单 / 进入交付前
- 输入：整个销售周期的会议、承诺、边界、风险
- 输出：Handoff Pack（承诺清单/边界/待复核/客户性格画像）
- 复核点：销售经理确认后正式提交
- 内核：Handoff Pack worker（商业版 certified）

### 4.5 开源/商业边界
| Skill | 开源（Helm Core） | 商业版（Pack A） |
|---|---|---|
| A1 | basic Meeting-to-Action worker（demo 级） | CRM 双向 + 草稿治理 + 历史复用 |
| A2 | basic shadow runtime（无优先级算法） | priority scorer + 客户健康度 + 主管视图 |
| A3 | 不开源 | 完全商业版 |
| A4 | 不开源 | 完全商业版 |

## 6. 数据接入
### 6.1 最小源
| 源 | 用途 | scope |
|---|---|---|
| 飞书/钉钉/企微会议 | 会议触发 + 纪要 | OAuth + 只读 |
| CRM | 商机/客户/活动 | API 只读 + 双向（需授权） |
| 邮箱 IMAP | 邮件历史 | 只读 |
| IM | 沟通历史 | OAuth + 只读 |

### 6.2 接入复杂度
| 系统 | 复杂度 | 工程师 |
|---|---|---|
| 标准 CRM | 低，1-2 天 | L1 |
| 定制 CRM | 中，3-7 天 | L2 |
| 自建/老旧系统 | 高，2-4 周 | L2/L3 + Helm 核心联运 |
| 邮箱 + 飞书/钉钉 | 低 | L1 |
| 多 BU / 多组织 | 高 | L2/L3 |

### 6.3 私有化
Helm Core 开源版支持；Pack A 商业版默认 SaaS；私有化需 Enterprise 合同 + Helm 核心评估；工程师不能擅自交付私有化（红线）。

## 7. 不做清单（红线）
| 不做 | 理由 |
|---|---|
| 自动发邮件给客户 | "推荐 vs 承诺"边界 |
| 自动写 CRM 字段 | 数据完整性 |
| 替销售做"承诺类"措辞 | CLAUDE.md 边界 |
| 自动判定成单/流单 | 销售决策权 |
| 替主管自动评分 | 不替代管理判断 |

**对外措辞红线：** 不得用"自动跟进"、"AI 替你成单"、"无需人工"。

## 8. 试点验收
| 指标 | 4 周 pilot | 6 月稳态 |
|---|---:|---:|
| 会后跟进延迟 | 2-4h → ≤30 分 | ≤15 分 |
| 跟进清单采纳率 | ≥60% | ≥80% |
| 主管介入有效率 | ≥40% | ≥60% |
| Handoff 完整度 | ≥3.5/5 | ≥4.2/5 |
| 越界承诺识别准 | ≥70% | ≥85% |
| 销售周报准备时间 | -50% | -70% |

## 9. 渠道与定价
**渠道：** 不上钉钉商城；GitHub → 公众号/知乎/掘金 → 认证工程师推荐。客户付费：与 Helm 直签商业版 + 与工程师独立服务合同（两条独立链路）。

| 项 | 价格 |
|---|---|
| Helm Core 开源 | ¥0 |
| Pack A Cloud | ¥29,800-99,800/年/工作区（30 席起） |
| Pack A Enterprise | ¥200,000+/年（私有化、SSO、SLA） |
| 工程师实施服务 | 市场化 |
| 4 周 paid POC | ¥50,000 |

## 10. 风险
1. CRM 厂商关系——可能视为竞品。**对策**：先打不依赖原生集成的子赛道
2. 会议接入隐私——录音/纪要含客户隐私。**对策**：默认"客户告知 + 销售确认"双门槛
3. 主管面板反弹——销售可能抗拒。**对策**：默认主管只看建议；销售可标记"私密会议"屏蔽
4. 首批 design partner ≥3 家是否就位——**待补充**
5. 认证工程师服务比例 ≥60% 目标——6 月节点验证
