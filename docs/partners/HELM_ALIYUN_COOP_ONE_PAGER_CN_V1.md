# Helm Aliyun Cooperation One-pager CN v1

## 1. Purpose

This document defines a reusable Aliyun cooperation narrative for Helm.

It is a partner-framing document only.
It does not claim any existing Aliyun commercial partnership unless separately documented elsewhere.

## 2. Helm’s Strongest Current Wedge

Helm 当前最强的切入口仍然是 **meeting-first operating wedge**：

- 会议把真实工作流带进系统
- dashboard 把会议后的推进优先级重新排清
- approvals 保留正式复核
- memory 把过程沉淀为经营记忆
- diagnostics 把 readiness 问题讲清楚

这让 Helm 已经不再只是会议记录工具，而是一条更完整的经营协同 loop。

## 3. Why Aliyun Is Relevant

阿里云相关能力和 Helm 的组合点主要在四层：

- 模型调用
- 长记忆与对象状态存储
- 审计 / 合规 /可控治理
- 联合企业解决方案交付

换句话说，Helm 更适合成为“高频工作流入口 + 经营记忆 + 正式复核”的应用层，而云侧提供更稳的基础设施与行业方案能力。

## 4. Model Invocation Fit

Helm 需要稳定的模型调用能力，但当前卖点不是模型本身，而是模型如何服务：

- 目标推进
- AI worker + human collaboration
- formal review
- memory accumulation

这与云侧的大模型调用能力是天然互补，而不是同一层价值。

## 5. Long-memory / Storage Fit

Helm 的长期壁垒之一是经营记忆，而不是一次性生成结果。

这意味着长记忆、对象状态、处理历史、审计线索和理由链会成为越来越重要的基础层诉求，适合与云侧存储和数据产品形成组合方案。

## 6. Audit / Compliance Fit

Helm 当前强调：

- recommendation 不等于 commitment
- formal review remains explicit
- governance posture remains visible
- no implied send authority
- no implied workflow control

这意味着 Helm 更适合进入那些需要“AI 帮忙推进，但关键动作必须可复核、可解释、可控”的企业场景。

## 7. Joint Enterprise Solution Framing

对外可以这样联合叙事：

**阿里云提供模型、存储、合规与行业解决方案底座，Helm 提供高频工作流入口、经营记忆与正式复核可读性。**

更具体地说：

- 会议、日程、邮箱、CRM 和内部系统把上下文带进 Helm
- Helm 把上下文变成推进链、经营记忆和复核面
- 云侧补足基础设施、审计与企业交付能力

## 8. What Is In Scope Now

当前可说清楚的范围：

- meeting-first operating wedge
- dashboard / meeting / approvals / memory / diagnostics 这条 loop
- self-serve trial + narrow billing lifecycle truth
- current China payment rail as narrow commercial rail
- future read-only connector strengthening narrative

## 9. What Is Intentionally Not Yet Claimed

当前不要对合作方 overclaim：

- no existing Aliyun commercial partnership claim
- no workflow engine claim
- no auto-send or auto-execution claim
- no broad connector platform claim
- no enterprise IAM / admin platform claim
- no finance console claim

## 10. Partner Closing Line

**Helm 适合作为企业高频工作流上的协同应用层，把入口、经营记忆和正式复核做深；云侧则提供模型、存储、审计和交付底座。**
