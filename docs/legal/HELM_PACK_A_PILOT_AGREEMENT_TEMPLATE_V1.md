# Helm Pack A 4 周 Paid Pilot 协议模板 V1

## 1. 文件状态与使用边界

本文件是 Pack A 第一个 design partner 的 **pilot agreement 起草模板**，用于减少首次起草时间。它不是法律意见，不是已签合同，不代表客户已经进入 pilot，也不替代客户或 Helm 法务终审。

使用本模板前必须完成：

1. 客户主体、签约主体、联系人、付款信息由创始人或 Helm 商务确认。
2. 数据处理协议（DPA）与本协议一并签署或作为附件签署。
3. 客户是否允许脱敏案例、公开 proof、cookbook 样例发布，必须单独勾选，不得默认同意。
4. 若使用境外模型 API、境外存储或境外协作工具，必须先完成跨境数据路径说明和法务复核。

## 2. 合同摘要

| 项 | 默认值 | 可调整项 |
|---|---|---|
| 服务名称 | Helm Pack A B2B SaaS 销售推进 4 周 paid pilot | 客户业务线名称 |
| 试点周期 | 4 周，另含 Week 0 启动准备 | 仅可在 SOW 中明确延长 |
| 试点费用 | ¥50,000，含 4 周 pilot 与基础交付支持 | 首个 design partner 可由创始人书面调整 |
| 服务形态 | Helm 官方实施团队交付；review-first；draft-only | 是否引入认证工程师需另列 |
| 目标 | 验证销售推进断层识别、Must Push、follow-up draft、manager review、handoff proof | 不承诺成交、续约、收入提升或客户外发成功 |
| 验收门 | Pack A runbook 默认 10 项中 ≥8 项通过 | 调整需写入 SOW |
| 数据保留 | pilot 结束后最长 90 天，除非法律、合同或导出争议需要保留 | DPA 优先 |

## 3. 签约方

甲方（客户）：`<客户公司全称>`

乙方（Helm）：`<Helm 签约主体全称>`

本协议中，甲方负责提供试点业务场景、样本数据、业务 owner 与 review 节奏；乙方负责提供 Helm Pack A pilot 服务、配置、人工复核支持、proof pack 和交付报告。

## 4. 服务范围

乙方在 4 周 paid pilot 中提供以下服务：

1. Pack A 默认配置启动：A1 meeting follow-up、A2 priority customers、A3 manager attention、A4 CS handoff pack。
2. 客户限定业务线的只读数据接入或手工样本导入评估。
3. 每周一次 review，复盘采纳率、误判、边界事件和下周调优项。
4. follow-up draft、Must Push、manager attention、handoff pack 的候选输出。
5. pilot proof pack：包括 baseline、样本量、指标变化、边界事件、客户主观评分和下一阶段建议。

乙方不提供以下服务，除非双方另签书面 SOW：

1. 自动外发邮件、IM、短信或客户可见消息。
2. 自动审批、自动改 CRM、自动改变交易阶段或自动承诺成交。
3. 完整私有化、SSO、SCIM、等保、专属 SLA 或完整安全审计。
4. 客户自研系统深度改造、历史数据全量迁移或大规模 ETL。
5. 法律、财务、税务、投资或人事决策建议。

## 5. 客户义务

甲方应在 Week 0 前完成：

1. 指定一名销售总监级以上 pilot owner。
2. 指定 5-15 名试点用户，建议包含销售代表、销售主管、CS lead。
3. 提供 3-5 场真实或脱敏会议样本、CRM 机会样本、handoff 样本和 baseline 指标。
4. 安排每周 review，确保 pilot owner 参加。
5. 确认哪些数据可接入、可脱敏、可进入 proof pack、可公开引用。
6. 配合法务和 IT 完成 DPA、访问授权、账号开通和最小必要数据导出。

如果甲方未能提供上述输入，乙方有权暂停启动，或将 pilot 改为 diagnostic-only，不承诺进入 4 周 paid pilot。

## 6. 乙方义务

乙方应：

1. 按最小必要原则处理甲方提供的数据。
2. 将高风险动作保持为 review-required，不自动执行。
3. 将 customer-facing 输出保持为 draft 或 candidate，不写成已发送、已审批、已承诺。
4. 在每周 review 中报告误判、边界事件和数据接入阻塞。
5. 在 pilot 结束时提供 proof pack 或 No-Go 复盘。
6. 按 DPA 要求完成数据导出、删除或保留说明。

## 7. 费用与付款

默认 pilot 费用为人民币 `¥50,000`。

建议付款结构：

| 时间 | 金额 | 条件 |
|---|---:|---|
| 协议签署后 5 个工作日内 | 100% | 作为 pilot 启动条件 |

可选结构：

| 时间 | 金额 | 条件 |
|---|---:|---|
| 协议签署后 5 个工作日内 | 50% | Week 0 启动准备 |
| Week 1 review 通过后 5 个工作日内 | 50% | 进入 Week 2-4 |

退款或中止：

1. 因甲方未提供必要数据、owner 或 review 节奏导致 pilot 无法启动的，乙方可扣除已发生启动成本，余额按双方书面确认退还。
2. 因乙方未能提供基础服务导致 pilot 无法继续的，双方按已消耗周期和实际交付内容协商退款。
3. 已完成 proof pack 或双方确认中止复盘后，pilot 费用原则上不再退还，除非协议另有约定。

## 8. 验收与结果表达

默认验收参考 [PACK_A_PILOT_RUNBOOK_V1.md](../sales/packs/PACK_A_PILOT_RUNBOOK_V1.md)：

1. 10 项作业质量指标中 ≥8 项通过，标记为 pilot pass。
2. Wrong Commitment Incident 必须为 0。
3. 高风险输出必须有 review-required 或 boundary note。
4. proof pack 必须标明 public-use posture：internal-only / anonymized / approved public。

边界表达：

1. 验收通过不等于客户承诺续约。
2. proof pack 不等于公开案例授权。
3. 系统建议不等于客户承诺。
4. follow-up draft 不等于已发送。
5. manager attention 不等于自动审批或自动干预。

## 9. 知识产权与交付物

1. Helm 平台、Pack A 模板、worker 设计、配置方法、文档和通用 know-how 归乙方或其授权方所有。
2. 甲方原始数据、客户资料、会议内容、CRM 记录和业务秘密归甲方或其授权方所有。
3. pilot 期间形成的客户特定配置、脱敏样例、proof pack 可按双方约定用于内部复盘。
4. 任何公开案例、公众号、知乎、媒体、ClawHub / cookbook 发布，必须有甲方单独书面授权。

## 10. 保密

双方应对在 pilot 中获知的商业秘密、客户资料、技术方案、价格、未公开产品路线和试点结果承担保密义务。

未经对方书面同意，不得向第三方披露：

1. 未脱敏客户名称、联系人、交易金额、客户名单。
2. 会议录音、会议原文、邮件原文、IM 原文。
3. CRM 明细、机会名称、报价单、合同条款。
4. proof pack 中标记为 internal-only 的内容。

## 11. 数据处理与安全

数据处理以 DPA 为准。若本协议与 DPA 冲突，以 DPA 中更严格的数据保护条款为准。

乙方默认采取：

1. 最小必要访问。
2. 工作区级隔离。
3. 访问权限限制。
4. 审计记录。
5. 去标识化或脱敏优先。
6. pilot 结束后最长 90 天保留，随后删除或归档为无法识别客户和个人的统计信息。

## 12. 跨境与模型 API 说明

若 pilot 使用境外模型 API、境外存储、境外日志或境外协作系统，乙方必须在启动前提供：

1. 涉及的数据类型。
2. 是否包含个人信息、敏感个人信息或重要数据。
3. 数据是否出境、出境路径、接收方和处理目的。
4. 是否使用脱敏、去标识化或摘要化输入。
5. 法务认为需要的个人信息影响评估、标准合同、认证或安全评估材料。

在该说明完成前，默认不将客户原始会议、CRM、邮件或 IM 内容发送至境外服务。

## 14. 中止与终止

任一方可在出现以下情况时书面要求暂停或终止：

1. 客户无法提供必要数据、owner 或 review 节奏。
2. 数据接入或合规限制导致 pilot 目标无法达成。
3. 任一方发生重大保密、数据安全或合规违约。
4. 连续 7 天以上关键连接器不可用，且双方无法达成替代方案。
5. 客户要求 Helm 执行自动外发、自动审批、自动承诺等违反 pilot 边界的动作。

终止后，双方应完成：

1. 未结费用结算。
2. 客户数据导出或删除。
3. proof pack 使用权限确认。
4. 公开材料撤回或保留确认。
