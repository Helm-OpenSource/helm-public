# Unified Detail Navigation Delivery Baseline Freeze Report

## 当前冻结的 delivery 资产范围

当前 unified detail navigation / cross-detail handoff 已正式进入：

- founder demo
- training cue
- acceptance cue
- delivery script
- retell template
- oral script

## 当前必须固定的 3 条链路讲法

### 1. proposal -> package -> customer-facing offer

- 最该看什么：
  - current node
  - current reason
  - handoff boundary
  - next action
- 最该判什么：
  - 当前是否已经值得从 shaping 切到 customer-visible 表达窗口
- 最该复用的话术：
  - “这不是普通跳页，而是从当前商业判断切到下一段更合适的 detail。”
  - “切页时 boundary、sendability 和 decision request 会一起移动。”

### 2. customer-facing offer -> external proposal -> reinforcement

- 最该看什么：
  - 当前是否 external-safe
  - 当前为什么要进入 reinforcement
  - 当前 strengthening 会不会放大 commitment 风险
- 最该判什么：
  - 当前是否应该继续增强表达，还是停在 review-before-send / boundary-only
- 最该复用的话术：
  - “Helm 先说明为什么现在适合切到下一页，再说明谁需要拍板。”
  - “stronger wording 不等于 commitment。”

### 3. package variants <-> reinforcement variants

- 最该看什么：
  - 当前 variant intent / stage / audience
  - 当前 strengthening level / fallback
  - 当前 boundary / non-commitment
- 最该判什么：
  - 当前到底应该继续 customer-visible、停在 internal-only，还是回退到 fallback
- 最该复用的话术：
  - “variant 和 strengthening 是一组连续判断，不是两个孤立配置页。”
  - “如果 strengthening 需要降级，variant 也要一起降级。”

## 页面原文与 delivery cue 的边界

### 属于页面原文

- Current node
- Previous / next detail
- handoff reason
- handoff boundary
- next action
- worker cue
- evidence drawer

### 只属于 training / oral / acceptance / delivery

- “这仍是第一轮 unified detail navigation baseline”
- “这不是 graph navigation platform、workflow engine 或 orchestration 平台”
- “当前只是把商业 detail 页收成连续经营链”

## 冻结结论

当前页面原文、training cue、acceptance template、retell template、oral script 和 delivery script 已经共用同一套 navigation / handoff / boundary / evidence 语义。

这意味着后续 founder demo、training、acceptance 和 delivery 不再需要各写一套“为什么现在切页”的说法。
