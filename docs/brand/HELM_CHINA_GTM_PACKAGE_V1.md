# Helm China GTM Package v1

## 1. Purpose

This document freezes the first complete China-market go-to-market package for Helm.

It translates the brand baseline into one reusable external package for:

- website and landing copy
- sales handoff
- founder demo
- trial conversion explanation
- partner conversations
- investor first-look narrative

It does not change product scope, route ownership, workflow authority, or send authority.

## 2. Official Category Name

**AI 经营协同操作系统**

This remains the only canonical public category name for Helm in the China market.

## 3. One-line Positioning

**Helm 让 AI 员工和团队围绕目标持续推进、正式复核，并把过程沉淀成可复用的经营记忆。**

## 4. Long Positioning Paragraph

**Helm 是一套 AI 经营协同操作系统。它把会议、日程、邮箱、CRM 和企业内部系统里的高频信号收成同一条推进链，让 AI 员工和团队成员始终清楚当前目标、为什么是现在、下一步是什么，以及哪些动作仍然必须停留在正式复核和可控治理之内，同时把过程沉淀成可追溯、可复用的经营记忆。**

## 5. Target Audience Framing

### 5.1 Users

Helm 帮一线成员把分散在会议、跟进、审批和对象状态里的上下文收成一条推进链，不再靠人肉记忆维持节奏。

### 5.2 Managers

Helm 让管理者一眼看懂当前最该推进什么、为什么是现在、关键动作是否仍需复核，以及团队已经沉淀了哪些经营记忆。

### 5.3 Enterprise Buyers

Helm 提供的是一套高频工作流驱动的经营协同工作台，而不是单点 AI 助手；它把推进、复核、记忆和治理放在同一套阅读顺序里。

### 5.4 Partners

Helm 适合与模型调用、长记忆存储、审计合规和行业方案交付结合，形成“高频工作流入口 + 经营记忆 + 正式复核”的联合方案。

### 5.5 Investors

Helm 从会议等高频工作流切入，先拿入口，再沉淀经营记忆，再补齐 AI 员工协同和治理边界，向更强的 business collaboration operating system 演进。

## 6. Core Value Pillars

### 6.1 Unified Context Entry

把会议、日程、邮箱、CRM 和内部系统的上下文统一进同一工作台。

### 6.2 Goal Progression

让 AI 员工先整理上下文、形成建议、理由链和下一步，再让人围绕关键动作做正式复核。

### 6.3 Operating Memory

把事实、承诺、阻碍、修正、对象状态和处理历史沉淀成可复用的经营记忆。

### 6.4 Formal Review

让关键动作先经过明确复核，而不是把建议误读成自动执行。

### 6.5 Controllable Governance

让团队一眼知道当前边界、责任、来源说明和风险，不把 Helm 说成自动控制台。

## 7. Key Proof Points From Current Product

Current proof should stay anchored in what current main already shows:

- dashboard already reads as `what matters now`
- meeting detail already reads as `why this meeting matters and what follows`
- approvals already reads as formal review and boundary surface
- memory already reads as writeback, confidence, lifecycle, ledger, and replay
- diagnostics already reads as workflow-scoped readiness, not whole-system readiness
- self-serve trial and billing lifecycle already exist with explicit `trialing / active / grace / read_only / canceled` truth
- current China payment rail is already framed as narrow checkout truth, not as a finance console

## 8. What Must Never Be Overclaimed

Helm must not be sold as:

- a meeting note taker only
- a generic AI platform
- an autonomous workflow engine
- an auto-send system
- an auto-commitment system
- a full enterprise governance platform
- a complete CRM / ERP / PM replacement

Public copy must keep these truths explicit:

- meetings are the strongest current wedge, not the whole product
- recommendation does not equal commitment
- formal review still exists
- governance remains visible
- key actions do not imply send authority or workflow control

### 9.1 Canonical Slogan

## 11. Voice and Tone Rules

### 11.1 What the voice should feel like

- clear
- commercially confident
- operational
- trustworthy
- governance-aware

### 11.2 What the voice should not feel like

- toy-like
- vague AI hype
- over-automated
- consumer companion language
- generic platform language

### 11.3 Copy rules

- speak in terms of goals, progression, review, memory, and boundary
- treat meetings as the strongest wedge, not the whole category
- prefer “推进、复核、记忆、治理” over “自动替你做完”
- keep claims anchored in current product proof
- separate “already proven” from “future path”
