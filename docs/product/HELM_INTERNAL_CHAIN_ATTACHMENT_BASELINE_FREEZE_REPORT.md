# Helm Internal Chain Attachment Baseline Freeze Report

## 冻结挂链关系

当前以下内容已经冻结为 internal operating workspace 的标准挂链方式：

- meetings
- decisions
- tasks
- retros / memory

## 挂到哪些对象

- lead / customer：商业会议、proposal / offer / follow-up 决策、审批任务、客户复盘
- candidate：interview meeting、fit judgement、next interview、offer readiness
- partner：partner meeting、fit / scope judgement、dependency risk、合作复盘
- workstream：跨对象的会议、审批、任务、复盘 summary

## 它们如何影响当前 judgement

- recent meetings：提供最新现场信号
- key decisions：说明当前为什么需要拍板或已经拍板
- next tasks：说明当前链条的推进动作
- retros / memory：说明为什么这个边界和判断要被保留

## 它们如何影响 next action 和 handoff

- next action 不再只是抽象建议，而是由 meeting / decision / task / retro 共同支撑
- handoff 不再只是角色标签，而是有证据、有上下文、有风险说明的接手

## 诚实边界

当前 attachment layer 仍是 summary-first：

- 不是完整 workflow engine
- 不是完整 meeting OS platform
- 不是完整 approval timeline platform
