# Helm Operating Constitution Baseline Freeze Report

## 已冻结的 Constitution truth

### Helm 是什么

Helm 是一个 workspace-first、membership-backed、judgement-first 的经营控制台。
它首先要帮助团队看清当前最重要的判断、最该推进的动作、最需要显式保留的边界。

### Helm 当前不是什么

- 不是完整战略平台
- 不是完整 knowledge platform
- 不是完整 workflow / orchestration engine
- 不是完整 company operating system
- 不是 generic chatbot 或 prompt playground

### Helm 追求什么结果

1. 让团队更快看清今天最重要的经营判断
2. 让 handoff、memory、boundary、approval 可追溯
3. 让 recommendation 能推动经营，但不越过 commitment 边界
4. 让 Helm 团队能开始在 Helm 上经营 Helm

### Helm 拒绝什么做法

1. 把 recommendation 伪装成 commitment
2. 把 boundary / prerequisite / dependency / non-commitment 隐到后台
3. 把 trial / paid 做成功能阉割版
4. 把 token / storage / retrieval 变成 customer-visible 单独计费项
5. 把 foundation 写成品牌口号或文化海报

### Helm 的经营边界

- workspace-first
- membership-backed
- controlled-trial
- lifecycle-based guard 必须 narrow、honest、可解释
- external communication 高风险默认需要 review / approval
- internal operating workspace 服务“让 Helm 团队在 Helm 上经营 Helm”

### Helm 默认判断优先级

1. 先判断当前主战役与经营压力
2. 再看 role / audience / handoff owner
3. 再看 replay / audit / memory / boundary trace
4. 最后才决定 action rail

## 已冻结的不可轻易改写 truth

1. workspace-first
2. membership-backed
3. controlled-trial
4. recommendation 不等于 commitment
5. discussion-only / boundary-only / review-before-send / non-commitment 必须显式可见
6. trial / paid 都开放完整当前核心功能
7. token / storage / retrieval 不对外单独计费
8. lifecycle-based guard 必须 narrow、honest、可解释

## 当前基线结论

Constitution Layer 当前已经足够冻结成正式 baseline。
后续页面、handoff、goal-driven 首页、customer success 深化与 role variants 扩展，都不应偏离这套 charter。
