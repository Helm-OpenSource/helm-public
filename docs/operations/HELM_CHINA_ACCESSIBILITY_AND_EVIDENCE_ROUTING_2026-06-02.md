## Two-Tier Evidence Model

Primary activation proof remains GitHub public-safe evidence.

| Tier | Counts As | Required Evidence | Does Not Count Alone |
| --- | --- | --- | --- |
| Primary proof | Primary activation proof | GitHub issue or discussion link with public-safe Golden Path result, first-change proof, blocker, docs friction, or integration use case | Stars, private chat, unlinked screenshots, private deployment notes |
| Assisted signal | Assisted activation signal | WeChat / QR / community handoff that later produces GitHub evidence, or an owner-approved redacted receipt | Private chat contact without a public-safe proof path |
| Reach-only signal | Reach/contact signal, not activation proof | README-visible contact discovery, WeChat first touch, or community entry without a GitHub link or owner-approved redacted receipt | Activation success, commercial intent, response obligation, or roadmap demand |

This prevents WeChat or community access from becoming a private,
unverifiable activation claim.

## Proof Collection Route

1. Point the person to the README Golden Path.
2. Ask them to change the synthetic `CASE-SAMPLE-002` fixture from
   `priorityScore: 64` to `priorityScore: 82`.
3. Ask them to rerun the README checks and observe whether the mapped signal
   changes from `info continue_followup` to `warning continue_followup`.
4. Route public-safe results:
   - Golden Path success, blocker, or friction: Golden Path issue template.
   - Docs ambiguity: documentation issue template.
   - Integration idea: integration request issue template with source object,
     data flow, and review boundary.
   - Broader scenario or question: Discussion #41.
5. If the person entered through WeChat, treat that as assisted access until it
   produces a GitHub link or owner-approved redacted receipt.

## Boundary Rules

Public channels and redacted receipts must not include:

- Real organization data.
- Real names, private contacts, private domains, internal IPs, or private URLs.
- Credentials, tokens, cookies, QR credentials, or secrets.
- Private environment details, production logs, or private implementation notes.
- Security vulnerability details; those must use `SECURITY.md`.
- Paid-offer, response-time, roadmap, private implementation, or obligation
  wording.
- Automatic external send, automatic write, automatic approval, execution, or
  settlement language.
