# HELM_PROACTIVE_DELIVERY_BASELINE_FREEZE_REPORT

## 冻结范围

当前 founder demo / training / acceptance / delivery 的主动机制基线，继续固定在以下页面和资产上：

- 首页
- opportunities
- approvals
- proposal / package 详情入口
- memory drill-down
- replay / audit / handoff / callback 下钻路径
- founder demo script
- manual acceptance paths
- delivery boundary
- active reporting cue
- collaboration cue
- decision request cue
- worker summary cue
- evidence drawer cue

## 每页最该看什么 / 最该判什么

### 首页

- 最该看：Helm 今天的经营判断、风险变化、第一动作、founder 决策请求
- 最该判：今天哪条线该成为唯一第一优先级
- 关键 cue：founder risk escalation、worker summary、decision request、evidence drawer

### opportunities

- 最该看：proposal / package 是否进入新阶段、当前 shaping window、下一步到底走跟进、会议还是澄清
- 最该判：这条机会的下一步 framing 是否安全
- 关键 cue：sales / delivery collaboration window、boundary summary、worker assignment、evidence drawer

### approvals

- 最该看：哪个 worker draft 已经成熟到需要 review
- 最该判：通过、改写、转人工还是继续拦住
- 关键 cue：review request、boundary summary、worker output、evidence drawer

## lifecycle / callback / authority cue

当前最关键的 authority cue 已固定为：

1. recommendation 不是 commitment
2. high-risk external moves 仍要先过 review / approval
3. replay / audit / memory 仍然是默认可下钻的解释层
4. callback continuity 与 ownership 仍然由 workspace / membership / approval actor 承接

## 哪些 cue 属于页面原文

- current judgement
- Helm already prepared
- decision request
- boundary summary
- worker summary
- evidence drawer

## 哪些 cue 只属于 training / oral / acceptance / delivery

- “Helm 不是把分析工作重新丢回给你”
- “先看变化、再看判断、再看拍板”
- “这是主动经营中枢，不是自动承诺平面”
- “Helm 可以先准备，但不会越权把 recommendation 说成 commitment”
