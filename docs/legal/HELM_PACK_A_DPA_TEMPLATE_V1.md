# Helm Pack A 数据处理协议模板 V1

## 1. 文件状态与角色假设

本文件是 Pack A 4 周 paid pilot 的数据处理协议（DPA）起草模板，不是法律意见，不是已签协议。

默认角色假设：

1. 客户是其 CRM、会议、邮件、IM、客户成功数据的个人信息处理者或数据控制方。
2. Helm 在 pilot 中按客户指示处理数据，通常处于受托处理方或处理服务提供方位置。
3. 具体法律角色、委托关系、共同处理关系、跨境路径和出境义务必须由法务按客户实际数据流确认。

## 2. 处理目的

Helm 仅为以下 pilot 目的处理客户数据：

1. 生成 Pack A Must Push、follow-up draft、manager attention、handoff pack 和 proof pack。
2. 验证销售推进断层识别、复核边界、作业质量指标和客户主观评分。
3. 调试连接器、字段映射、阈值、词典和候选输出质量。
4. 按客户要求导出、删除或归还 pilot 数据。

Helm 不得将客户数据用于：

1. 与 Pack A pilot 无关的模型训练。
2. 向其他客户展示未授权样例。
3. 公开案例、媒体材料、cookbook、ClawHub 发布，除非客户另行书面授权。
4. 自动发送、自动审批、自动改 CRM 或自动承诺客户结果。

## 3. 数据类别

允许在最小必要范围内处理：

| 类别 | 示例 | 默认处理姿态 |
|---|---|---|
| 账号与组织信息 | 试点用户姓名、角色、企业邮箱、团队归属 | 仅用于账号、权限和 review |
| CRM 元数据 | 机会阶段、owner、预计金额区间、下一步、预计关闭日期 | 优先字段级脱敏或区间化 |
| 会议材料 | 会议摘要、行动项、承诺候选、阻塞项 | 优先摘要，不默认保留原文录音 |
| 邮件 / IM 样本 | 跟进状态、客户等待信号、handoff 片段 | 优先片段化和去标识化 |
| 系统日志 | 登录、导入、worker 运行、错误和审计事件 | 不写入明文客户敏感内容 |
| proof 数据 | 指标变化、样例截图、客户主观评分 | 标记 public-use posture |

默认禁止处理，除非客户和法务书面批准：

1. 身份证件、银行卡、家庭住址、健康、定位、未成年人等敏感个人信息。
2. 客户生产密钥、access token、cookie、私钥、数据库密码。
3. 国家秘密、重要数据、核心数据或客户明确标记不得出域的数据。
4. 与销售推进无关的大规模原始邮箱、IM、录音、CRM 全量历史数据。

## 4. 处理期限与保留

默认保留期：

1. pilot 期间：为服务交付所必需的周期内保留。
2. pilot 结束后：最长 90 天，用于 proof pack、争议处理、导出和删除确认。
3. 90 天后：删除可识别客户、个人或具体机会的数据；仅可保留匿名化统计或不可逆聚合指标。

客户可在 pilot 结束时选择：

1. 导出全部 pilot 输出。
2. 删除全部可识别数据。
3. 保留脱敏 proof pack。
4. 单独授权公开案例或 cookbook 样例。

## 5. 安全措施

Helm 应按最小权限和最小必要原则执行：

1. 工作区级隔离，不跨客户 workspace 查询。
2. 试点用户最小权限授权，按角色区分销售、主管、CS、管理员。
3. 生产密钥、令牌和连接器凭据不得写入文档或日志。
4. 审计记录覆盖导入、运行、导出、删除和高风险候选输出。
5. 对外部模型或第三方服务传输前优先脱敏、去标识化、摘要化。
6. 发生数据安全事件时，按合同约定时限通知客户并采取补救措施。

## 6. 子处理方与第三方服务

Helm 如使用云服务、模型 API、日志、监控、邮件或协作工具作为子处理方，应在启动前列明：

| 子处理方 | 服务用途 | 数据类别 | 地域 | 是否出境 | DPA / 安全材料 |
|---|---|---|---|---|---|
| `<待填写>` | `<待填写>` | `<待填写>` | `<待填写>` | 是 / 否 | `<链接或附件>` |

未经客户书面确认，Helm 不得把客户原始会议、CRM、邮件或 IM 内容发送给未列明的第三方。

## 7. 跨境数据流动

如果 pilot 全程使用中国境内基础设施、境内模型和境内协作工具，则本节记录为“不涉及出境”。

如果涉及境外模型 API、境外云服务、境外日志或境外人员访问，应在启动前完成：

1. 出境目的、方式、范围、数据种类、接收方和保留期限说明。
2. 是否包含个人信息、敏感个人信息或重要数据的评估。
3. 是否适用个人信息出境标准合同、个人信息保护认证或数据出境安全评估的法律判断。
4. 是否需要个人单独同意、客户员工告知、客户侧 privacy notice 或内部制度补充。
5. 境外服务不可用、撤回授权、客户终止时的数据删除路径。

默认红线：在本节未完成前，不将客户原始会议、CRM、邮件或 IM 内容发送到境外服务。

## 8. 客户权利与个人请求协助

客户负责面对其员工、联系人、客户和最终个人的信息权利请求。Helm 在合理范围内协助客户：

1. 查询、复制、导出 pilot 中与个人相关的数据。
2. 更正、删除或停止处理可识别个人的数据。
3. 解释自动化或算法候选输出的处理逻辑边界。
4. 处理撤回同意、拒绝处理或投诉请求。

Helm 不直接代表客户向个人作出法律承诺，除非双方另有书面约定。

## 9. 数据删除与销毁证明

pilot 结束、终止或客户提出删除请求后，Helm 应在约定期限内：

1. 删除客户可识别原始数据、临时导入文件、缓存和导出副本。
2. 删除不再必要的日志字段或将其匿名化。
3. 保留必要的安全审计日志，但不得包含明文客户敏感内容。
4. 向客户提供删除确认或销毁证明。

建议删除完成期限：`15 个工作日内`，如客户或法务要求可调整。

## 10. 安全事件通知

发生或可能发生个人信息泄露、篡改、丢失、越权访问或未授权出境时，Helm 应：

1. 立即采取补救措施。
2. 在双方约定时限内通知客户。
3. 提供事件类别、影响范围、已采取措施、建议客户采取的措施和联系人。
4. 配合客户依法履行监管报告、个人通知或审计要求。

建议通知时限：`发现后 24 小时内初报，72 小时内补充初步调查结果`，最终以法务确认版本为准。

## 11. 审计与记录

Helm 应保留以下记录：

1. 数据接入清单。
2. 子处理方清单。
3. 关键权限变更。
4. 导入、导出、删除记录。
5. 高风险候选输出和人工复核记录。
6. proof pack public-use posture。

客户可按合同约定要求查看与其 pilot 有关的安全和处理记录。查看范围不得暴露其他客户、Helm 商业秘密或系统安全敏感信息。

## 12. 与 Pack A 边界的关系

本 DPA 不扩大 Helm 执行权。即使客户提供数据授权，也不代表 Helm 获得：

1. 自动外发权限。
2. 自动审批权限。
3. 自动修改 CRM 或客户系统权限。
4. 自动承诺成交、续约、回款或交付结果权限。
5. 公开客户案例、客户名称、销售对话或 cookbook 样例的权限。

所有 customer-facing 输出默认是 draft 或 candidate，高风险动作默认 review-required。
