# Pack A 候选验证电话脚本（30 分钟）

## 1. 使用边界

本脚本用于创始人给 Top 2-3 Pack A design partner 候选打 30 分钟验证电话。它不是销售承诺、不是实施启动、不是合同谈判。

必须遵守：

1. 不索取真实客户名单、会议原文、邮箱原文、CRM 明细进入公开仓库。
2. 不承诺自动外发、自动审批、自动改 CRM、自动成交或收入提升。
3. 不把候选口头兴趣写成 pilot 承诺。
4. 只记录候选代号和脱敏判断，真实客户信息留在创始人私有笔记。

## 2. 会前准备

会前 10 分钟完成：

1. 用 [PACK_A_DESIGN_PARTNER_EVALUATION_V1.md](./PACK_A_DESIGN_PARTNER_EVALUATION_V1.md) 给候选纸面打分。
2. 准备候选代号：`A / B / C` 或 `SaaS-A / SaaS-B / SaaS-C`。
3. 确认本次只验证三件事：配合度、公开 proof、数据接入复杂度。
4. 准备一句话定位：Helm Pack A 是 B2B SaaS 销售推进控制台，不是 CRM、会议纪要工具或自动销售机器人。

## 3. 开场（0-3 分钟）

推荐话术：

> 今天这通电话不是正式销售，也不是项目启动。我们正在选择 Pack A 第一个 design partner，核心是验证 Helm 能不能在 4 周内把“会后跟进、主管介入、客户等待、CS handoff”压成可复核的 Must Push 和 proof。今天不会要求你提供敏感客户数据，也不会承诺定制功能或自动执行。

必须主动补边界：

> Helm 第一阶段只做建议、草稿、复核和 proof，不做自动外发、自动审批、自动改 CRM，也不承诺成交结果。

## 4. 关键验证问题（3-24 分钟）

### 4.1 销售推进痛点（3-8 分钟）

1. 过去 30 天里，最常见的销售推进断层是什么？
2. 断层更常发生在会后、报价后、试用后、客户等待时，还是 CS handoff 后？
3. 如果系统每天只提示 3 件最该推进的事，你希望是哪三类？
4. 这些事现在由谁盯：销售本人、销售主管、CS，还是创始人？

强信号：

- 对方能立刻举出最近 2-3 个推进卡住的真实类型。
- 销售总监或 VP 直接参与痛点描述。
- 痛点涉及收入、续约、客户等待或管理者介入，而不是泛泛“提高效率”。

弱信号：

- 只说“想试试 AI”。
- 没有明确 owner 或业务线。
- 主要问题是 BI 报表、客服工单或项目管理。

### 4.2 配合度真实性（8-14 分钟）

直接问：

> 如果进入 4 周 pilot，我们要求每周 1 次 45-60 分钟 review，销售总监级 owner 必须参加，试点团队 5-15 人。你们能保证吗？

追问：

1. 谁是 pilot owner？
2. owner 能不能每周参加 review？
3. 是否接受创始人或 Helm 工程 lead 在 Day-1 陪跑？
4. 是否接受第一阶段先用 Pack A 默认配置，不先做深度定制？

判断：

| 回答 | 判断 |
|---|---|
| 当场给出 owner 和时间窗口 | 强候选 |
| 说“我回去问问销售总监” | 候选池，需二次确认 |
| 只想先免费试试，不承诺 owner | No-Go |

### 4.3 可公开 proof（14-19 分钟）

直接问：

> 如果 4 周 pilot 顺利，我们希望 6 周内发布一篇脱敏案例，可能包含匿名指标、流程图和少量脱敏样例。你们方法务或品牌侧能接受吗？

追问：

1. 是否允许匿名指标，例如 follow-up 延迟下降、漏跟进数量、handoff 完整度？
2. 是否允许脱敏截图或重写样例？
3. 是否允许公众号 / 知乎文章只写行业、规模、问题，不写公司名？
4. 哪些内容绝不能公开？

判断：

| 回答 | 判断 |
|---|---|
| 明确可做脱敏 proof，并知道内部审批人 | 强候选 |
| “应该可以，但要法务看” | 可候选，需 Week 0 前确认 |
| 完全不允许任何 proof 或指标 | 不适合作为第一个 design partner |

### 4.4 数据接入复杂度（19-24 分钟）

直接问：

> 你们 CRM 是标准版还是深度定制？会议、IM、邮箱、CRM 里哪些可以在一周内用只读方式或手工导出样本？

追问：

1. CRM 是销售易、纷享销客、HubSpot、Salesforce、自研，还是轻量表格？
2. 会议纪要来自飞书、钉钉、企微、腾讯会议、Zoom，还是人工记录？
3. 邮箱和 IM 是否能提供脱敏样本？
4. IT 或法务是否会要求先走安全评估、DPA、内网部署？

判断：

| 回答 | 判断 |
|---|---|
| 标准 CRM + 可导出样本 + owner 能协调 | 强候选 |
| 自研系统但可手工导出样本 | 可候选 |
| 数据不能出域、也不能手工脱敏导出 | 不适合作为首个 pilot |

## 5. 收口（24-30 分钟）

推荐话术：

> 我们会把今天信息只用于候选评估，不会把贵司真实信息写入公开文档。下一步如果双方都认为合适，我们会发一版 4 周 paid pilot scope、DPA 和 Week 0 checklist，再决定是否进入启动准备。

必须确认：

1. 是否愿意进入下一通 45 分钟 scope call。
2. 谁参加：业务 owner、IT / 法务联系人、销售总监或 VP。
3. 是否可以在 1 周内确认数据接入边界。
4. 是否接受 paid pilot，而不是无限期免费 PoC。

## 6. 会后 5 分钟打分

会后只做脱敏记录：

| 项 | 判断 |
|---|---|
| 候选代号 | A / B / C |
| 业务痛点 | strong / medium / weak |
| 配合度 | strong / medium / weak |
| proof 公开 | yes / maybe / no |
| 数据接入 | low / medium / high |
| 商业路径 | paid pilot / diagnostic then paid / defer |
| 决策 | Top 1 / Candidate Pool / Nurture / No-Go |

## 7. 红线

以下情况默认不进第一个 Pack A pilot：

1. 只想免费试用，不愿指定 owner。
2. 要求 Helm 自动外发、自动改 CRM 或自动承诺成交。
3. 不能提供任何会议、CRM、邮件或 handoff 样本。
4. 完全拒绝脱敏 proof，且不允许内部指标复盘。
5. 关键问题不是销售推进断层。
