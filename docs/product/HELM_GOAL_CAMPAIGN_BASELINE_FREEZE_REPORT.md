# Helm Goal / Campaign Baseline Freeze Report

## 已冻结的 Goal / Campaign 定义

### 当前目标层

目标层不是 OKR / KPI 平台，而是当前经营主线的 product-grade summary 层。

### 当前 campaign 层

campaign 用来回答：

1. 当前公司在打哪场仗
2. 当前最重要的推进链是什么
3. 当前最重要的 3 个判断是什么
4. 当前最重要的阻塞是什么
5. 当前最应该推进什么

## 当前 Goal / Campaign 与其他层的关系

- 与 detail chain 的关系：决定当前 detail 服务的是哪条主战役
- 与 handoff 的关系：决定哪个 role 现在最应该接手
- 与 role 的关系：决定 Founder / Sales / Delivery / Customer Success / Recruiting / Partner 当前被拉入哪条主线
- 与 memory 的关系：决定当前 judgement 需要哪些 replay / audit / trace 来支撑

## 当前产品入口引用方式

### 首页 / dashboard

首页当前应把 Current Campaign、Top 3 Operating Judgements、Top 3 Chain Moves、Top 3 Blockers、Top 3 Decision Requests 前置。

### settings / billing overview

settings 继续把 goal / campaign 和 lifecycle / seat / entitlement truth 放在同一组织 operating 语境里。

### handoff / detail

handoff 继续用 goal / campaign 解释“为什么现在轮到这个 role 接手”。

## 当前公司在打哪场仗

当前公司在打的是一场“让 Helm 团队开始真正用 Helm 经营 Helm，并把这套 operating truth 持续挂回销售、交付、客户成功、招聘、伙伴与内部决策链”的主战役。

## 当前最重要的推进链

1. internal operating workspace 主链
2. customer success issue / escalation / renewal risk 主链
3. dashboard goal-driven operating surface 主链

## 当前最重要的 3 个判断

1. 当前应该先保哪条经营链
2. 当前哪个阻塞最值得拍板
3. 当前该由哪个 role 接手什么

## 当前最重要的阻塞

1. 如果首页不把 campaign / blockers 前置，团队仍会回到对象入口堆叠
2. 如果 role / audience / memory / goal baseline 不冻结，后续扩展容易漂移
3. 如果 boundary 不继续显式可见，recommendation / commitment 会重新混在一起

## 当前最应该推进什么

先冻结 baseline，再把 dashboard 真正重组为 goal-driven home，然后再继续更细的 customer success 深化与 role variants 扩展。

## 当前基线结论

Goal / Campaign Layer 当前已经足够冻结。
它的职责不是充当 slogan，而是继续组织首页 summary、priority、blockers、decision request 和 handoff priority。
