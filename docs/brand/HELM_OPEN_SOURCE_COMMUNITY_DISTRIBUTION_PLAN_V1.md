# Helm 开源 + 社区 + 自媒体分发计划 V1

## 1. 目的
建立 **Helm Core 开源 → 开发者/实施工程师社区 → 大厂高人气社区 → 公众号自媒体放大** 四级传播链路，6 个月内形成可验证市场声量与认证工程师供给。

**核心定调：** GitHub Discussions + 大厂高人气社区为主 + 公众号自媒体放大传播。

## 2. 范围与边界
**做：** 四级链路设计；候选社区清单；首发素材；6 个月目标。
**不做：** 付费投放（除非单独裁决）；钉钉应用市场；SEO/搜索广告；海外社区（本期聚焦中国）。

## 3. 四级链路
```
Layer 1：GitHub（开发者一级入口）
Layer 2：大厂高人气社区（开发者/实施工程师转化）
Layer 3：公众号自媒体（业务买家放大）
Layer 4：线下与圈层活动（高净值客户，半年后启动）
```

## 4. Layer 1：GitHub 运营
### 4.1 仓库结构
| 仓库 | 内容 | 受众 |
|---|---|---|
| helm-ai/helm-core | runtime、basic workers、SDK、demo | 开发者、技术评审 |
| helm-ai/helm-docs | 双语文档（中 + 英） | 全部 |
| helm-ai/helm-pack-template | Pack 开发模板（不含商业 Pack 实现） | 实施工程师、伙伴 |

**说明：** 三个商业 Pack 实现不开源（与 commercial boundary plan §6 对齐）；Pack 模板与 Cookbook 公开。

## 6. Layer 3：公众号自媒体
### 6.1 Helm 自有公众号
**定位：** 业务买家与决策者入口，不做技术深度。
**方向：** 客户案例（脱敏）、Pack 发布与价值、认证工程师名录、创始人观点。
**节奏：** 每周 ≥1 篇，每月 ≥1 篇深度。

### 6.2 行业自媒体合作清单（候选）
| 类型 | 候选 | 适配 |
|---|---|---|
| 科技媒体 | 36Kr、量子位、机器之心、AI 前线、智能涌现 | 全 Pack |
| To B 行业媒体 | ToB 行业头条、牛透社、SaaS 点评网、崔牛会 | Pack A 首选 |
| 跨境电商媒体 | 雨果跨境、亿邦动力 | Pack C |
| 咨询/SI 圈层 | LinkedIn 中国、咨询群 | Pack B |

**合作形式：** 优先稿件投递（免费）；联合直播（0-5 万/场）；不做硬广（除非裁决）。

### 6.3 关键约束
- 文案必须经"建议 vs 承诺"边界审核（不得用"自动完成"、"AI 替你做"、"无需人工"）
- 提到商业能力必须注明"商业版"
- 提到认证工程师必须注明"独立服务方"

## 9. 风险
1. GitHub 国内访问——Gitee/Coding 镜像
2. 公众号触达天花板——质量优先
3. 工程师内容贡献意愿低——L2/L3 续期要求 ≥1 篇公开内容（非硬指标）
4. 媒体改稿夸大——合同约定文案需 Helm 终审
5. Layer 4 启动时机——6 个月内确认
