# Release Readiness Receipts

本目录只保存不含 secret 的 release readiness receipt 元数据。

当前唯一支持的 machine-readable receipt 是 `mirror-clean:<receipt-id>`。

推荐使用命令生成 receipt，而不是手写 JSON：

```bash
npm run public-mirror:clean-receipt -- \
  --receipt-id public-mirror-2026-05-17-smoke \
  --source-ref main@abcdef0 \
  --mirror-root <candidate>
```

该命令会先运行 public mirror build / preflight / verify，只有通过后才写 receipt。

生成后可单独校验 receipt，再进入完整 release gate：

```bash
npm run public-mirror:clean-receipt:check -- \
  --receipt mirror-clean:public-mirror-2026-05-17-smoke
```

Receipt 约束：

- 文件路径：`docs/operations/release-readiness-receipts/<receipt-id>.json`
- `kind` 必须是 `public-mirror-clean`
- `receiptId` 必须与 env var 中的 `<receipt-id>` 一致
- `commandEvidence` 必须至少包含一条 exitCode 为 `0` 的 `public-mirror:build` 或 `public-mirror:verify` 命令
- JSON 不得包含 secret、客户数据、本机绝对路径或 raw logs

示例模板：

```json
{
  "receiptId": "public-mirror-2026-05-17-smoke",
  "kind": "public-mirror-clean",
  "createdAt": "2026-05-17",
  "sourceRef": "branch-or-commit-without-secret-values",
  "commandEvidence": [
    {
      "command": "npm run public-mirror:build -- --mirror-root <candidate>",
      "completedAt": "2026-05-17",
      "exitCode": 0,
      "scannedFiles": 2963
    }
  ],
  "notes": "Generated by scripts/build-public-mirror-clean-receipt.ts. Do not include secrets, credentials, customer data, local absolute paths, or raw logs."
}
```
